## Requisitos Funcionais

### Inserir Números

- O sistema deve permitir a entrada de números de 0 até 9.

### Oprações básicas

- O sistema deve permitir operações matemáticas como: Soma, Subtração, Multiplicação, Divisão, Porcentagem.

### Exibição de Resultado

- O sistema deve calcular e exibir o resultado após o acionamento do botão "=".

### Exibição da expressão

- O sistema deve exibir a expressão matemática em construção antes do resultado final.

### Limpeza parcial

- O sistema deve permitir apagar o último caractere digitado.

### Limpeza total

- O sistema deve permitir limpar completamente a operação atual.

### Controle de operadores

- O sistema deve impedir a inserção consecutiva de operadores inválidos. (ex: ++).

### Números decimais

- O sistema deve permitir entrada de valores decimais através do separador decimal.

## Requisitos Não Funcionais

### Usabilidade

- A interface deve ser simples e intuitiva.

### Responsividade visual

- Os botões devem apresentar feedback visual ao serem pressionados ( Quando clicacos fazer animação de clique ).

### Manutenibilidade

- O código deve ser organizado de forma modular, separando lógica de cálculo e interface.

## Regras de Negócio

- O cálculo só deve ser realizado após o acionamento do botão "=".
- O resultado exibido pode ser usado como base para uma nova operação.
- A expressão deve seguir uma sequência válida de números e operadores.
- O sistema deve impedir operações matemáticas inválidas durante o uso.
- Ao limpar a operação, a expressão e o resultado devem ser reiniciados.
